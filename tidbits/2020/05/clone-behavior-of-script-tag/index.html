Clone Behavior of <script> Tag

Wolfram Kriesing - May 11, 2020 - tagged with: #web #javascript #DOM #knowledgebase

I wanted to scriptNode.cloneNode(true) and expected it to re-evaluate the JS of the node, but it didn't.

Each <script> element has a property flag called "already started".

This flag is already true, so a clone won't reevaluate the script. Solution: copy it manually.

Read it all on https://stackoverflow.com/questions/28771542/why-dont-clonenode-script-tags-execute/28771829#28771829
